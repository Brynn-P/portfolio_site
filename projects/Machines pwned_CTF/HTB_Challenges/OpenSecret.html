Open Secret

This is my first ever public write up on any box/CTF/challenge, I hope that you all enjoy and are able to take something away from this.

Please note that the flag is not visible here, learning is not done through copy and paste ;-)

OpenSecret is a Hack The Box challenge classified as very easy

This was a fun JWT challenge to do and a good one to have when you're looking for a quick bit of fun

The challenge description reads “A simple help desk portal where users can submit support tickets. The application uses JWT tokens for session management, but something seems off about how they're implemented. Can you find the security flaw?”

And we are off to the races!

For the challenge site this time, it was found at 154.57.164.64:32115

To navigate to this in the browser, simply use http:// with the IP address in your preferred browser

For pretty much anything I touch involving web, my first stop is always burp suite(although I do need to explore zap a bit)

Now that I have it open and I see the page, The obvious first step is to try submit a ticket and see what happens but looking through what is captured in Burp Suit

What we received back does not seem too exciting but every little thing can be a clue, we received an error in the browser stating no session token provided.

This is when I decided to take a peek under the hood to see what was going on

You can do this in a few ways but I use Ctrl + Shift + i to pull up the developer tools

While I was browsing through the HTML and JS, I spotted something that all developers fear, and vibecoding sometimes produced lol, I had found that the secret key was hardcoded in plaintext in the script tag of the client side code.

This is the security flaw we were looking for, with this, anyone with access to the page can extract the key and forge valid tokens

And there we have it, the flag HTB{0***_***_***_**s }, which is hidden because I believe in you, you can find it yourself

Happy Hacking :-)
